   - 突破关键阻力位且成交量放大

   - 单币种不超过总资金的20%

1. **止盈卖出**：

   - NOX：$0.00075

   - 其他币种按各自止损位执行

3. **割肉止损**：
   - 亏损超过50%且无明显反弹迹象

#### 动态调整
1. **根据舆情调整**：
   - 马斯克利好 → 上调DOGE/VIRTUAL止盈位或加仓
   - 特朗普政策利空 → 下调止损位或减仓避险

2. **根据技术分析调整**：
   - 突破关键阻力位 → 上调止盈位
   - 跌破关键支撑位 → 下调止损位或止损

3. **根据市场情绪调整**：
   - 恐慌贪婪指数极度恐惧 → 谨慎操作，保留现金
   - 恐慌贪婪指数极度贪婪 → 考虑部分止盈

## 报告格式（详细规范）

### 7个部分及格式要求

#### 1. 💼 持仓概况
**格式要求：**
- 标题：### 1. 💼 持仓概况
- 必须标注数据来源（Bankr / 本地计算 / Rainbow）
- 表格列：币种、数量、成本价(USDC/币)、现价(USDC/币)、市值(USDC)、盈亏(USDC)、盈亏(%)、状态
- **关键：成本价和现价必须是每个币的单价，不是总成本**
- 必须有小计行
- 表格后添加汇总信息：总投资 | 当前总市值 | 总盈亏

**示例：**
```
| 币种 | 数量 | 成本价(USDC) | 现价(USDC) | 市值(USDC) | 盈亏(USDC) | 盈亏(%) | 状态 |
|------|------|-------------|-----------|-----------|-----------|--------|------|
| ETH | 0.0101 | 1,186.14 | 1,996.47 | 20.16 | +8.18 | +68.3% | holding |
...
| **小计** | - | - | - | **54.46** | **+1.31** | **+2.4%** | - |

**总投入**: 53.65 USDC | **当前总市值**: 54.46 USDC | **总盈亏**: +1.31 USDC (+2.4%)
```

#### 2. 🔒 当前订单 + 交易计划
**格式要求：**
- 标题：### 2. 🔒 当前订单 + 交易计划
- 表格列：币种、操作类型、目标价格、数量、状态
- 包含止盈、止损订单
- 标注已执行的交易

**示例：**
```
| 币种 | 操作类型 | 目标价格 | 数量 | 状态 |
|------|----------|----------|------|------|
| ETH | 止盈 | 2,350 | 0.0101 | 待触发 |
| ETH | 止损 | 1,850 | 0.0101 | 待触发 |
```

#### 3. 📈 市场概况
**格式要求：**
- 标题：### 3. 📈 市场概况（日期时间 北京时间）
- 表格列：币种、价格(USDC)、24h涨跌、分析
- 必须包含：BTC、ETH、SOL、DOGE
- 可选：VIRTUAL、BNKR等持仓币种
- 添加市场总结段落

**示例：**
```
| 币种 | 价格(USDC) | 24h涨跌 | 分析 |
|------|-----------|---------|------|
| BTC | 68,880 | +0.45% | 企稳反弹... |
| ETH | 1,996.47 | +1.86% | 突破2000关口... |
...

**市场总结：** 大盘整体...（分析段落）
```

#### 4. 📰 舆情摘要
**格式要求：**
- 标题：### 4. 📰 舆情摘要
- 编号列表3-5条
- **必须包含马斯克和特朗普相关言论**
- 每条格式：标题 — 内容（影响分析）

**示例：**
```
1. **马斯克确认X平台将集成加密货币功能** — 2月15日马斯克确认...（利好）
2. **特朗普加密政策面临市场考验** — 尽管特朗普承诺...（利空）
3. **ETH技术面显示上涨潜力** — 分析师预测ETH月底...（利好）
```

#### 5. 🎯 交易决策（动态评估）
**格式要求：**
- 标题：### 5. 🎯 交易决策（动态评估）
- 子标题：**当前状态** — 总持仓、总盈亏、可用USDC、目标进度
- 子标题：**市场分析** — 基于舆情和价格的分析
- 子标题：**建议操作** — 具体买入/卖出建议，包含价格和数量
- 子标题：**风险提示** — 止损提醒、仓位控制

#### 6. 📋 今日操作记录
**格式要求：**
- 标题：### 6. 📋 今日操作记录
- 表格列：时间、操作、币种、数量、价格、价值、盈亏
- 记录所有已执行的交易
- 添加日盈亏统计

**示例：**
```
| 时间 | 操作 | 币种 | 数量 | 价格 | 价值 | 盈亏 |
|------|------|------|------|------|------|------|
| 12:35 | 卖出 | NOX | 2,700 | 0.000902 | $2.44 | 止损 |

**日盈亏统计：** +$X.XX（持仓浮盈）
```

#### 7. 结尾
**格式要求：**
- 标题：### 7. 结尾（或 ## 7. 📌 总结）
- 必须包含：
  - 下次报告时间（北京时间）
  - 数据来源
  - 日盈亏统计（目标/当前/止损线）

**示例：**
```
**下次报告时间：** 2026年2月17日 22:30（北京时间）

**数据来源：** Bankr API / 本地持仓计算

**日盈亏统计：**
- 目标盈利：+10 USDC
- 止损限制：-7 USDC
- 当前盈亏：+1.31 USDC
- 进度：13%
```

### 表格格式规范（16:09更新）

**必须使用标准Markdown表格格式：**

```markdown
| 列1 | 列2 | 列3 | 列4 |
|-----|-----|-----|-----|
| 数据1 | 数据2 | 数据3 | 数据4 |
| 数据5 | 数据6 | 数据7 | 数据8 |
```

**关键要求：**
1. ✅ 使用 `|` 作为列分隔符
2. ✅ 表头行后必须有分隔行 `|---|---|---|`
3. ✅ 每行数据用 `|` 包裹
4. ✅ 对齐方式：默认左对齐，数字可右对齐
5. ✅ 禁止用空格或制表符模拟表格

**错误示例（禁止）：**
```
币种        数量          价格
--------------------------------
ETH         0.0101        1987
```

**正确示例（必须）：**
```markdown
| 币种 | 数量 | 价格(USDC) |
|------|------|-----------|
| ETH | 0.0101 | 1,987.22 |
| BTC | 0.0005 | 68,408 |
```

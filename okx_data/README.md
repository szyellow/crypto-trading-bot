# AI 自主加密货币交易系统 v4.1

一个基于 OKX 交易所 API 的智能加密货币交易机器人，采用多 Agent 架构，整合**币市麻雀战法**、多维度共振策略、情绪分析和动态风控系统。

## 🎯 币市麻雀战法 v4.1 (时区感知版)

> **核心理念**: "麻雀虽小，五脏俱全" - 追求小而稳的盈利，见好就收，积少成多。

### 策略特点
- **日目标**: $9 (1%盈利目标)
- **周目标**: $21 (2.5%盈利目标)
- **止盈**: +3% (麻雀见好就收)
- **止损**: -0.5% (严格止损)
- **持仓时间**: 15分钟-2小时 (快进快出)
- **单笔仓位**: $12-$15 (时区动态调整)
- **时区感知**: 6个交易时段动态调整策略参数

## 🚀 主要特性

### 币市麻雀战法 v4.1 (核心策略)

#### 时区感知系统

| **欧盘早盘** | 08:00-12:00 | 高波动 | 仓位$15-18，持仓15-45分钟 ⭐⭐⭐⭐⭐ |
| **欧盘午盘** | 12:00-16:00 | 最高波动 | 仓位$15-20，持仓10-30分钟 ⭐⭐⭐⭐⭐ |
| **美盘早盘** | 16:00-20:00 | 高波动 | 仓位$15-18，持仓15-45分钟 ⭐⭐⭐⭐⭐ |
| **美盘午盘** | 20:00-00:00 | 中等波动 | 仓位$12-15，持仓15-60分钟 ⭐⭐⭐⭐ |

#### 动态止盈止损
根据趋势评分自动调整止盈比例：

| 趋势评分 | 止损 | 止盈 | 策略 |
|---------|------|------|------|
| ≤5分 | -1% | 2% | 严格止损 |
| 6-7分 | -1.5% | 3% | 标准策略 |
| 8-9分 | -2% | 10% | 放宽止盈 |
| ≥10分 | -3% | 15% | 趋势跟随 |

#### 多维度共振策略
四维共振建仓系统：

| 维度 | 权重 | 数据源 | 阈值 |
|------|------|--------|------|
| **舆情评分** | 30% | CoinGecko + 新闻情绪 | ≥7分 |
| **技术面** | 25% | RSI/MA5/成交量/波动率 | ≥6分 |
| **资金流向** | 25% | 持仓量变化 + 成交量异动 | 资金流入 |
| **大盘环境** | 20% | BTC/ETH趋势 + 资金费率 | 良好 |

### 其他特性
- **金字塔建仓**: 分层买入策略
- **阴线买入**: 连续2根阴线+趋势≥6分时触发
- **横盘暂停**: 趋势3-5分且波动率<0.5%时暂停
- **暴跌反弹**: 24h跌幅>10%且趋势回升时抄底
- **智能趋势分析**: 基于多维度技术指标的趋势评分

### 情绪分析 (v2.3+)
- **CoinGecko 集成**: 实时获取市场情绪数据
- **新闻情绪分析**: 通过新闻 API 分析币种相关新闻的看涨/看跌情绪
- **Sub-agent 模式**: 独立服务处理情绪数据，避免主程序阻塞

### 数据提醒系统 (v2.4+)
- **自动数据检查**: 确保每次报告都基于最新数据
- **报告时间戳追踪**: 防止重复显示旧数据
- **数据获取清单**: 自动检查所有 API 数据是否已更新

### 风险控制
- **智能止损**: 趋势≤5分→-1%, 6-7分→-1.5%, ≥8分→-2%
- **黑名单管理**: 自动将止损币种加入黑名单，趋势回升后自动解除
- **稳定币过滤**: 自动跳过 USDC、USDT 等稳定币
- **紧急停止**: 支持 EMERGENCY_STOP.flag 文件紧急停止系统

## 🛠️ 安装与配置

### 1. 安装依赖
```bash
cd okx_data
npm install
```

### 2. 配置 OKX API
在 `okx-api.js` 中配置你的 API 密钥：
```javascript
const API_KEY = 'your_api_key';
const API_SECRET = 'your_api_secret';
const PASSPHRASE = 'your_passphrase';
```

### 3. 启动情绪数据服务 (可选但推荐)
```bash
node sentiment-client.js
```

## 📊 策略配置

### 币市麻雀战法配置 (config-sparrow.js)

```javascript
module.exports = {
  // 核心参数
  dailyTarget: 9,           // 日目标 $9 (1%)
  weeklyTarget: 21,         // 周目标 $21 (2.5%)
  takeProfit: 0.03,         // 止盈 3%
  stopLoss: -0.005,         // 止损 -0.5%
  
  // 时区感知配置
  timezoneAware: true,
  tradingSessions: {
    asiaEarly:  { start: 0,  end: 4,  intensity: 2, position: [10, 12], holdTime: [30, 90] },
    asiaLate:   { start: 4,  end: 8,  intensity: 5, position: [12, 15], holdTime: [15, 60] },
    europeEarly:{ start: 8,  end: 12, intensity: 5, position: [15, 18], holdTime: [15, 45] },
    europeLate: { start: 12, end: 16, intensity: 5, position: [15, 20], holdTime: [10, 30] },
    usEarly:    { start: 16, end: 20, intensity: 5, position: [15, 18], holdTime: [15, 45] },
    usLate:     { start: 20, end: 24, intensity: 4, position: [12, 15], holdTime: [15, 60] }
  },
  
  // 动态止盈配置
  dynamicTakeProfit: {
    trend5: { stopLoss: -0.01, takeProfit: 0.02 },   // 趋势≤5分
    trend67:{ stopLoss: -0.015, takeProfit: 0.03 },  // 趋势6-7分
    trend89:{ stopLoss: -0.02, takeProfit: 0.10 },   // 趋势8-9分
    trend10:{ stopLoss: -0.03, takeProfit: 0.15 }    // 趋势≥10分
  }
};
```

### 多维度共振策略

#### 建仓条件
必须同时满足以下四个维度：

| 维度 | 权重 | 数据源 | 通过标准 |
|------|------|--------|----------|
| **舆情评分** | 30% | CoinGecko情绪 + 新闻情绪 | ≥7分 |
| **技术面** | 25% | RSI/MA5/成交量/波动率 | ≥6分 |
| **资金流向** | 25% | 持仓量变化 + 成交量异动 | 资金流入良好 |
| **大盘环境** | 20% | BTC/ETH趋势 + 资金费率 | BTC≥5分, ETH≥5分 |

**综合共振分数**: 加权平均 ≥7分 方可建仓

#### 动态仓位管理
根据共振分数自动调整仓位：

| 共振分数 | 建议仓位 | 说明 |
|---------|---------|------|
| 9-10分 | $40 | 共振强烈，重仓出击 |
| 8-9分 | $32 | 共振良好，标准仓位 |
| 7-8分 | $25 | 共振一般，基础仓位 |
| <7分 | $0 | 共振不足，禁止建仓 |

### 选股门槛
- 趋势评分 ≥ 7分 (可配置)
- 24h 涨跌幅 -5% ~ 15%
- 成交量 ≥ 500万 USDT
- 价格 > $0.1

### 止盈策略
- 盈利 1.5% → 减仓 25%
- 盈利 3% → 止盈 50%
- 盈利 5% → 清仓

### 动态止损
| 趋势评分 | 止损比例 |
|---------|---------|
| ≤5分    | -1%     |
| 6-7分   | -1.5%   |
| ≥8分    | -2%     |

## 📈 监控报告

系统每 5 分钟生成一次交易检查报告，包含：
- 账户总资产和持仓概况
- 市场扫描结果（通过筛选的币种）
- AI 交易决策（买入/卖出/HOLD）
- 趋势分析和情绪数据

## 🔄 策略自迭代

系统自动分析交易数据并优化策略参数：
- 胜率 < 30% → 收紧止损 20%
- 盈亏比 < 1 → 降低止盈
- 持仓时间 > 6小时 → 降低选股门槛
- 连续3次止损 → 降低单笔金额 20%
- 现金 < 30% → 增加最大持仓

## 🚨 紧急停止

创建 `EMERGENCY_STOP.flag` 文件可立即停止系统：
```bash
touch EMERGENCY_STOP.flag
```

删除该文件后可重新启动：
```bash
rm EMERGENCY_STOP.flag
```

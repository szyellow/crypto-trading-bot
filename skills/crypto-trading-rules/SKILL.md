# Crypto Trading Rules Skill

## 交易规则清单

### 1. 黑名单规则

#### 稳定币 (永久禁止)
- USDC
- USDT
- USDG
- DAI
- TUSD
- BUSD
- **规则**: 永久禁止买入，不需要释放机制

#### 止损币种 (条件释放)
- 触发止损后自动加入
- **解除条件** (满足任一即可):
  - 1次趋势评分 ≥ 9分 (直接解锁)
  - 或连续2次趋势评分 ≥ 8分
  - 或连续3次趋势评分 ≥ 7分
- 例如: NEAR (当前4/10分)

### 2. 买入条件检查清单

#### 必须同时满足 (全部条件):
1. ✅ **不在黑名单** (稳定币/止损币种/手动禁止)
2. ✅ **趋势评分 ≥ 7分**
3. ✅ **非横盘状态** (趋势3-5分且波动率<0.5%为横盘，禁止买入)
4. ✅ **持仓占比 < 阈值** (根据趋势评分动态调整)
   - 趋势≥8分: 阈值20%
   - 趋势6-7分: 阈值15%
   - 趋势≤5分: 阈值10%
5. ✅ **冷却期已结束** (分层冷却期)
   - 趋势10分: 15分钟
   - 趋势8-9分: 20分钟
   - 趋势6-7分: 30分钟
6. ✅ **波动率 ≥ 0.5%** (优选1.5%)
7. ✅ **24h涨跌在合理范围** (-5% ~ 15%)
8. ✅ **价格 > $0.1** (过滤低价币)
9. ✅ **成交量 ≥ 500万USDT** (确保流动性)
10. ✅ **近期未买入** (避免频繁交易同一币种)

#### 额外买入策略:
- **阴线买入**: 连续2根阴线 + 趋势≥6分 + 价格<MA5
- **暴跌反弹**: 24h跌幅>10% + 趋势回升至≥6分
- **金字塔建仓**: 首仓$25 → 跌10%补仓$15 → 再跌10%补仓$10

### 3. 卖出条件

#### 止盈:
- 动态调整: 趋势8-10分 → 10-15%
- 趋势下降时下调止盈

#### 止损:
- 趋势10分: -3%
- 趋势8-9分: -2%
- 趋势6-7分: -1.5%
- 趋势≤5分: -1%

### 4. 仓位管理

- 单笔最大: $40
- 最大持仓: 5个
- 现金储备: 保持30%以上

### 5. 每日检查清单

- [ ] 检查黑名单是否需要更新
- [ ] 检查止损线是否正常
- [ ] 检查持仓是否超仓
- [ ] 检查止盈单是否正确挂单

### 6. 数据准确性规则

#### 每次检查和执行操作前必须:
1. ✅ **获取实时数据** - 从OKX API获取最新价格、持仓、账户信息
2. ✅ **验证数据准确性** - 对比本地记录和API返回数据
3. ✅ **检查数据时效性** - 确保数据不是缓存的旧数据
4. ✅ **确认连接状态** - 确保API连接正常，无网络问题

#### 数据获取优先级:
1. **账户余额** - 总资产、可用USDT
2. **持仓信息** - 各币种数量、成本价
3. **市场价格** - 当前价格、24h涨跌
4. **订单状态** - 止盈单、止损单是否正常

#### 数据异常处理:
- 如果API获取失败 → 重试3次
- 如果数据不一致 → 以API数据为准，更新本地记录
- 如果网络中断 → 暂停操作，等待恢复

#### 重要提醒:
- **OKX API有缓存**: 数据延迟约30-60秒，这是正常的
- 不要过分关注秒级价格变化
- 以实际成交价格为准

**稳定币处理**:
- 发现稳定币被买入 → 立即卖出释放资金
- 稳定币占用资金无法增值

**横盘期处理**:
- 趋势3-5分 + 波动率<0.5% = 横盘
- 横盘期禁止买入

**止损线修复**:
- 收紧: -2.5% → -2.0% (更容易触发)
- 放宽: -2.5% → -3.0% (更难触发)

### 7. 实时数据强制规则 (重要!)

#### 每次发送交易检查报告时必须:
1. ✅ **执行实时命令** - 必须调用 `exec` 执行 `node ai_trading_bot.js`
2. ✅ **获取最新数据** - 从OKX API获取实时账户、持仓、价格数据
3. ✅ **基于实时数据决策** - 所有交易决策必须基于刚获取的实时数据
4. ✅ **禁止复制粘贴** - 严禁使用旧数据或缓存数据回复
5. ✅ **显示数据时间戳** - 报告必须显示数据获取时间

#### 执行流程:
```
收到交易检查提醒
    ↓
调用 exec: cd /root/.openclaw/workspace/okx_data && node ai_trading_bot.js
    ↓
等待执行完成，获取实时输出
    ↓
解析实时数据（总资产、持仓、盈亏、趋势评分）
    ↓
基于实时数据生成报告和决策
    ↓
发送实时报告
```

#### 违规后果:
- 使用旧数据 → 错误决策 → 亏损
- 复制粘贴 → 数据滞后 → 错过机会
- 不执行命令 → 虚假报告 → 失去信任

**记住: 没有实时数据，就没有交易决策！**
